Exception Types

In Python, exceptions represent errors that occur during the execution of a program. There are many built-in exception types, each representing a specific kind of error. Here's an overview of some common exception types:

1. KeyError

When it occurs: Raised when a dictionary key is accessed that does not exist.

data = {"name": "Alice"}
print(data["age"])  # Raises KeyError because 'age' is not a key in the dictionary

2. ValueError

When it occurs: Raised when a function receives an argument of the correct type but an inappropriate value.

num = int("abc")  # Raises ValueError because "abc" cannot be converted to an integer

3. TypeError

When it occurs: Raised when an operation or function is applied to an object of inappropriate type.

result = "5" + 5  # Raises TypeError because you can't add a string to an integer

4. IndexError

When it occurs: Raised when trying to access an index that is out of the valid range for a list, tuple, or similar sequence.

my_list = [1, 2, 3]
print(my_list[5])  # Raises IndexError because index 5 is out of range

5. AttributeError

When it occurs: Raised when an attribute reference or assignment fails, typically when the object does not have that attribute.

my_list = [1, 2, 3]
my_list.append()  # Raises AttributeError because append requires an argument

6. NameError

When it occurs: Raised when a local or global name is not found.

print(x)  # Raises NameError because 'x' is not defined

7. FileNotFoundError

When it occurs: Raised when trying to open a file that does not exist.

with open("nonexistent_file.txt", "r") as file:
    data = file.read()  # Raises FileNotFoundError

8. ZeroDivisionError

When it occurs: Raised when attempting to divide a number by zero.

result = 10 / 0  # Raises ZeroDivisionError because division by zero is undefined

9. ImportError

When it occurs: Raised when an import statement fails to find the module or import specific functions/classes from the module.

import nonexistent_module  # Raises ImportError

10. IOError

When it occurs: Raised when an input/output operation (like reading or writing a file) fails for an I/O-related reason.

Note: Starting from Python 3.3, IOError is merged into OSError.

with open("somefile.txt", "r") as file:
    data = file.read()  # Could raise IOError if something goes wrong during I/O

11. RuntimeError

When it occurs: Raised when an error occurs that doesn’t fall into any other category. Often used for errors related to incorrect program logic.

def my_func():
    raise RuntimeError("This is a runtime error")

my_func()  # Raises RuntimeError with a custom message

12. StopIteration

When it occurs: Raised to signal the end of an iteration, typically used by iterators to let the for loop know when to stop.

my_iter = iter([1, 2, 3])
while True:
    print(next(my_iter))  # Raises StopIteration when the iterator is exhausted

13. AssertionError

When it occurs: Raised when an assert statement fails.

x = 5
assert x > 10, "x is not greater than 10"  # Raises AssertionError

14. OSError

When it occurs: Raised for system-related errors such as file system issues, failed system calls, or out-of-memory conditions.

import os
os.remove("non_existent_file.txt")  # Raises OSError if the file does not exist

15. MemoryError

When it occurs: Raised when an operation runs out of memory, typically when trying to create a huge data structure or perform a resource-heavy operation.

huge_list = [0] * 10**10  # Might raise MemoryError if the system runs out of memory

Custom Exception Handling

You can also create your own custom exceptions by subclassing the built-in Exception class:

class CustomError(Exception):
    pass

def my_function():
    raise CustomError("This is a custom error")

try:
    my_function()
except CustomError as e:
    print(f"Caught custom error: {e}")
